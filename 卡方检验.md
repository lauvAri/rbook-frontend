## 脚本名称

卡方检验

## 描述

测试卡方检验

## R脚本代码

```R
# =======================================================
# 脚本名称: 卡方检验 (原始明细数据版)
# =======================================================

# --- [变量注入] ---
# 对应原始代码: correct = FALSE
# 如果后端未注入，默认为 FALSE
if (!exists("use_correction")) use_correction <- FALSE 

# --- [数据读取与清洗] ---
if (exists("input_file") && file.exists(input_file)) {
  print(paste("正在读取原始数据文件:", input_file))
  
  # 1. 读取数据
  # stringsAsFactors = TRUE 有助于 table() 函数正确处理分类变量
  data <- read.csv(input_file, stringsAsFactors = TRUE)
  
  # 2. 校验列数
  if (ncol(data) < 2) {
    stop("数据格式错误: CSV 文件至少需要包含两列数据 (例如: 分组, 结果)")
  }
  
  # 3. 提取前两列 (通用化处理)
  # 这样做的好处是: 无论用户 CSV 的列头叫 "group/x" 还是 "性别/是否治愈"，代码都能跑
  col1 <- data[, 1]
  col2 <- data[, 2]
  
  print(paste("分析变量 1:", names(data)[1]))
  print(paste("分析变量 2:", names(data)[2]))
  
} else {
  # --- [模拟数据] ---
  # 如果没文件，生成模拟的原始数据，用于演示
  print("未检测到输入文件，生成模拟明细数据...")
  set.seed(123)
  col1 <- sample(c("Treat", "Control"), 100, replace = TRUE)
  col2 <- sample(c("Cured", "NotCured"), 100, replace = TRUE)
}

# --- [核心逻辑: 对应原始代码的 Table <- table(...) ] ---
# 统计频数表 (Contingency Table)
Table <- table(col1, col2)

print("=== 列联表统计 (Counts) ===")
print(Table)

# --- [核心逻辑: 对应原始代码的 chisq.test(...) ] ---
res <- chisq.test(Table, correct = as.logical(use_correction))

print("=== 卡方检验结果 ===")
print(res)

# 虽然不需要可视化，但输出期望值矩阵是判断卡方检验是否合法的关键
print("=== 期望频数 (Expected) ===")
print(res$expected)

# 简单的文本结论输出
if(res$p.value < 0.05) {
  print(">>> 结论: 在 0.05 显著性水平下，拒绝原假设，认为两个变量存在显著关联。")
} else {
  print(">>> 结论: 在 0.05 显著性水平下，无法拒绝原假设，尚不能认为两个变量存在关联。")
}
```

## 自定义参数

**参数1**：

变量名：use_correction

显示标签：连续性校正

类型：BOOLEAN

默认值：false

说明：是否应用 Yates 连续性校正 (correct参数)
